## 📂 Completed Rooms & Key Learnings

### 1. Fundamental Security Models & Principles
* **Room Link:** [Security Principles](https://tryhackme.com/room/securityprinciples)
* **Core Concepts Mastered:**
  * **CIA Triad:** Understood the core pillars of security—*Confidentiality, Integrity, and Availability*—and mapped real-world incident scenarios to each domain.
  * **DAD Triad:** Analyzed the adversarial counterparts—*Disclosure, Alteration, and Denial/Destruction*.
  * **Security Models:** Explored access control frameworks like **Bell-LaPadula** (Confidentiality focus: *No Read Up, No Write Down*) and **Biba** (Integrity focus: *No Read Down, No Write Up*).
  * **ISO/IEC 19249:** Applied design principles including *Least Privilege*, *Attack Surface Minimisation*, and *Error/Exception Handling*.

### 2. Data Encoding & Representation
* **Room Links:** [Data Representation](https://tryhackme.com/room/datarepresentation) & [Data Encoding](https://tryhackme.com/room/dataencoding)
* **Core Concepts Mastered:**
  * **Hexadecimal & Decimal Conversions:** Mastered base-16 conversions manually and via tools (e.g., resolving hex value scales up to millions).
  * **ASCII & Unicode Frameworks:** Decoded alphanumeric representations, control characters (like `BEL`), and modern multi-byte character sets (UTF-16/UTF-32).

### 3. Core Security Engineering Responsibilities
* **Room Link:** [Introduction to Security Engineering](https://tryhackme.com/module/introduction-to-security-engineering)
* **Core Concepts Mastered:**
  * **Secure by Design:** Embraced the philosophy that baking security into the initial design yields the highest Return on Investment (ROI).
  * **Risk & Policy Exceptions:** Analyzed how business continuity and organizational needs necessitate managed policy exceptions.
  * **Human Factor & Change Management:** Acknowledged humans as the weakest link and explored structured change management to keep security aligned with evolving infrastructures.
